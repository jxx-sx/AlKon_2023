## 커밋 규칙

`solve : 문제 이름` - 문제 첫 solve  
`틀린이유 : 문제 이름` - 틀린 소스코드 제출  
`refactor : 문제 이름` - solve 이후 코드 또는 알고리즘 수정  
`docs : 파일명` - 문서 수정

### 예시

- solve : A+B
- 틀렸습니다 : A+B
- 런타임 에러 (Segfault) : A+B
- 시간 초과 : A+B
- refactor : A+B
- docs : README
